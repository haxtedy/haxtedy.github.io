前端的一些面试点

1.输入完网址按下回车，到看到网页这个过程中发生了什么？

  • 域名解析：DNS（Domain Name Server，域名服务器）将域名解析为对应的IP地址。
  • 发起TCP的3次握手，建立客户端与服务器的连接。
  • 客户端发起http请求。
  • 服务器响应http请求，返回请求页面的html代码。
  • 浏览器得到html代码后，解析html代码，并请求html代码中的资源。
  • 服务器再次返回请求资源。
  • 浏览器得到所需的所有资源，对页面进行渲染，呈现给用户。

3.Get与Post的区别

  • get重点在从服务器上获取资源；post重点在向服务器发送数据；
  • get传输数据是通过URL请求，以field（字段）= value的形式，置于URL后，并用”?”连接，多个请求数据间用”&”连接，如http://127.0.0.1/Test/login.action?name=admin&password=admin，这个过程用户是可见的； post传输数据通过Http的post机制，将字段与对应值封存在请求实体中发送给服务器，这个过程对用户是不可见的；
  • Get传输的数据量小，因为受URL长度限制，但效率较高；Post可以传输大量数据，所以上传文件时只能用Post方式；
  • get是不安全的，因为URL是可见的，可能会泄露私密信息，如密码等； post较get安全性较高；
  • get方式只能支持ASCII字符，向服务器传的中文字符可能会乱码。post支持标准字符集，可以正确传递中文字符。

4.浏览器的缓存机制

参考：https://www.jianshu.com/p/54cc04190252

缓存机制：判断本地缓存是否过期，如果没有过期，直接使用，否则使用协商缓存。

强缓存：强缓存生效返回200，与协商缓存失效的返回一样，但是它来自于from disk cache或from memory cache

Expires(http/1.0)：判断缓存是否过期。缺陷：存的是服务器端的时间，与客户端时间有出入。

Cache-Control(http/1.1)：包含很多指令，比如max-age;no-cache;no-store等。max-age中保存的是持续时间而不是时间点，这是优于Expires的地方。

协商缓存：向服务器发送请求，判断本地缓存是否被更新过，没有就直接使用本地缓存，更新了，服务器需要发送新的内容。

Last-Modified 和If-Modified-Since：根据时间的改动。Last-Modified服务器发送给客户端，资源内容最后被更新的时间，会保存在本地缓存中。浏览器发送请求时，会附带If-Modified-Since，也就是本地缓存之前保存的Last-Modified，如果服务器端的Last-Modified和If-Modified-Since相等，说明没有更改，可以直接使用本地缓存，返回304；否则返回200，并且发送新的内容。缺陷：只能以秒计时，另外本地打开缓存文件，会修改Last-Modified。

ETag和If-None-Match：根据内容的改动。ETag是资源内容的一个唯一标识，由服务器计算产生，当资源发生变化，ETag就会重新生成。服务器发送给浏览器资源及其ETag。当浏览器发送请求时，会将ETag放到If-None-Match中，如果If-None-Match和服务器端的ETag匹配上了，就返回304，否则返回200，并且发送新的内容。
